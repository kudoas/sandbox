実装する split コマンドの仕様

要件
- [] split を参考にするが要件は自分で決定して良い
- option を実装する
  - [] -l: Create split files line_count lines in length.
  - [] -n: Split file into chunk_count smaller files.
  - [] -b: Create split files byte_count bytes in length.
- [] イレギュラーな入力などにもできるだけ対応する
- [] 大きなファイルを扱えるようにする
- [] 実行速度が速くなる工夫を書く
- [] go test で実行できるテストコードを書く
- [] Goの標準ライブラリか自作したパッケージのみ使用可能
- [] ファイルの分け方は自由

man split
DESCRIPTION
     The split utility reads the given file and breaks it up into files of 1000 lines each (if no
     options are specified), leaving the file unchanged.  If file is a single dash (‘-’) or absent,
     split reads from the standard input.
     - -l 1000 がデフォルト
- I: ファイル
- O: 例外の時はメッセージ、それ以外はない（ファイルを分割する）


If additional arguments are specified, the first is used as the name of the input file which is to be split.  If a second additional argument is specified, it is used as a prefix for the names of the files into which the file is split.  In this case, each file into which the file is split is named by the prefix followed by a lexically ordered suffix using suffix_length characters in the
range “a-z”.  If -a is not specified, two letters are used as the suffix.

If the prefix argument is not specified, the file is split into lexically ordered files named with the prefix “x” and with suffixes as above.
